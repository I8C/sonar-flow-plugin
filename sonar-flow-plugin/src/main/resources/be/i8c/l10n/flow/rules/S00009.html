There should always be a branching condition defined in the properties of a BRANCH step. Either in the switch property or in the labels of the child steps.

If the switch property of a BRANCH step is not defined, then the 'evaluate labels' property should be set to true. This indicates that the branching conditions are defined in the labels of the child step. If the switch is defined then the 'evaluate labels' property should be false or null.

Noncompliant Code Example

  Any service with a branch step that doesn't have the label expression set to 'true' while no switch value has been given.

Compliant Solution

  Give a switch value or set the label expression value to 'true'.
